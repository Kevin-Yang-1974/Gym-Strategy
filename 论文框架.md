#论文框架建议

##1 问题重述与背景分析

本节用于说明赛题背景、研究对象和本文需要解决的核心问题。

建议包括：

1. 智能健身房运营背景；
2. 积分促活机制的意义；
3.题目中的核心矛盾：
 - 提高用户线下锻炼活跃度；
 -促进线上分享与互动；
 - 提升会员续费和私教购买；
 - 控制积分成本；
 - 保证平台经营收益；
4. 本文研究目标：
 - 构建用户状态和行为特征；
 -预测会员续费和私教购买概率；
 -设计分层积分策略；
 - 在经营约束下选择综合表现最优的候选策略。

##2 模型假设与符号说明

本节用于明确模型适用边界，并统一全文符号。

###2.1 模型假设

建议包括：

1.会员商品有效期可以根据商品名称、价格和商品分类规则近似确定；
2. 用户在会员到期后 $H$ 天内再次购买会员类商品，视为窗口内续费；
3. 到期后已经完整观察 $H$ 天但未续费的用户，视为窗口内未续费或高流失风险样本；
4. 到期后尚未观察满 $H$ 天且未续费的用户，作为真正右删失样本处理；
5. 历史数据中缺少真实积分干预记录，因此积分效果属于情景模拟结果，而非真实实验因果估计；
6. 积分主要通过改变用户活跃度和互动强度影响会员续费概率与私教购买概率；
7. 若采用 Cox 比例风险模型，需假设解释变量对续费风险率的相对影响在观察期内基本稳定；
8. 同一用户多个会员周期样本在控制行为和消费特征后可近似处理，或在模型求解时进行稳健性说明。

###3.2 商品分类与权益周期识别

根据商品名称、价格和订单信息识别：

- 连续包月；
- 普通包月；
- 季卡；
- 年卡或长期卡；
- 社区优惠卡；
- 健身卡；
-体验卡；
- 次卡；
- 团购卡；
- 活动卡；
-赠送权益；
- 优惠套餐；
- 私教体验课或单次私教课；
- 私教小课包；
- 私教中课包；
- 私教大课包；
- 未知商品类。

由此得到会员权益结束时间：

$$
EndDate_i=PayDate_i+Duration_i
$$

其中，$PayDate_i$ 表示会员产品支付日期，$Duration_i$ 表示该会员产品对应的权益时长。

商品分类和权益周期识别建议遵循以下规则：

| 商品类别 |识别依据 | Duration / Sessions处理 |
|---|---|---|
| 连续包月 | 商品名含连续包月、自动续费等关键词 | 每期按30天处理 |
| 普通包月 | 商品名含月卡、包月等关键词 | 按30天处理 |
| 季卡 | 商品名含季卡、三个月等关键词 | 按90天处理 |
| 年卡或长期卡 | 商品名含年卡、半年卡、长期卡等关键词 | 半年卡按180天，年卡按365天，其他长期卡按名称识别 |
| 社区优惠卡 | 商品名含社区优惠等关键词 |归入优惠类会员产品，按名称中的月/季/年识别周期 |
| 健身卡 | 商品名泛称健身卡 |继续根据月/季/年/次等关键词细分，不能直接统一按月卡处理 |
|体验卡 | 商品名含体验卡、体验券等关键词 | 按1/3/7/14天等名称信息识别，无法识别时列入待核验 |
| 次卡 | 商品名含次卡、次数卡等关键词 |作为次数型入场权益，不强行转为固定天数会员 |
| 团购卡、活动卡 | 商品名含团购、活动等关键词 |归入优惠类会员产品，用于构造 $M_{i,t}^{discount}$，周期按名称和价格规则识别 |
|赠送权益 | 商品名含赠送、0元等关键词 | 不作为正常收入订单，可用于判断权益状态，需单独标记说明 |
| 私教体验课或单次私教课 | 商品名含私教体验、单次私教等关键词 | $Sessions=1$ |
| 私教小/中/大课包 | 商品名含私教、课包、节、次等关键词 | 优先从“X节”“X次”提取课时；无法提取时按小/中/大课包区间估计 |
| 未知商品类 | 无法可靠识别的商品 | 不强行赋予 $Duration$ 或 $Sessions$，保留人工核验 |

优惠类商品不额外新增未定义变量，而是用于构造消费结构特征中的 $M_{i,t}^{discount}$。

###3.3 用户行为特征构造

构造以下特征：

1.线下活跃度 $A_{i,t}$；
2.线上互动强度 $S_{i,t}$；
3. 消费结构特征 $M_{i,t}$；
4. 流失风险特征 $L_{i,t}$；
5.会员权益状态 $ActiveMember_{i,t}$ 与 $DTE_{i,t}$；
6. 私教课包状态 $PT_{i,t}$。

###3.4续费与私教购买标签构造

构造会员续费标签：

$$
Y_i^{renew,H}=\begin{cases}
1, & NextBuyDate_i-EndDate_i\leq H \\
0, & C-EndDate_i\geq H \text{ 且 } (NextBuyDate_i-EndDate_i>H \text{ 或未观察到下一次会员购买}) \\
\text{censored}, & C-EndDate_i<H \text{ 且未观察到续费}
\end{cases}
$$

构造私教购买标签：

$$
Y_{i,t}^{pt}=\begin{cases}
1, & \text{用户在预测窗口内购买私教产品}\\
0, & \text{否则}
\end{cases}
$$

##4 模型建立

本节用于系统建立各个子模型及其衔接关系。

###4.1 用户状态划分与状态转移分析

定义用户状态：

$$
Z_{i,t}=g(A_{i,t},S_{i,t},M_{i,t},L_{i,t},PT_{i,t},ActiveMember_{i,t})
$$

估计自然状态转移概率：

$$
P_{ab}^{0}=P(Z_{i,t+1}=b|Z_{i,t}=a)
$$

该部分主要用于解释用户状态自然演化规律，并为分层积分策略提供依据，不直接作为收益预测的核心计算模块。

###4.2会员续费生存模型

围绕会员到期日建立有限观察窗口下的续费生存模型。

Cox 模型形式为：

$$
h_i(t)=h_0(t)\exp(\beta^TX_i)
$$

其中，$h_i(t)$ 表示用户 $i$ 在到期后第 $t$ 天续费的风险率，$X_i$ 为用户特征向量。

用户在到期后 $H$ 天内续费的概率为：

$$
\hat{p}_i^{renew,H}=P(T_i\leq H|X_i)=1-S_i(H)
$$

其中，$H=30$ 天可作为主分析窗口，$H=60$ 天可作为稳健性分析窗口。

###4.3 私教购买模型

定义预测窗口内是否购买私教产品：

$$
Y_{i,t}^{pt}=\begin{cases}
1, & \text{用户在预测窗口内购买私教产品}\\
0, & \text{否则}
\end{cases}
$$

可建立 Logistic 模型：

$$
P(Y_{i,t}^{pt}=1|X_{i,t}^{pt})=\frac{1}{1+\exp[-(\phi_0+\phi^TX_{i,t}^{pt})]}
$$

其中，$X_{i,t}^{pt}$ 包括活跃度、无氧训练量、历史消费、私教购买历史、课包次数和私教单价等变量。

###4.4 积分响应模型

设积分策略 $r$ 下用户获得积分为：

$$
Point_{i,t}(r)=\min\left(p_0N_{i,t}^{eff}+p_1I_{i,t}^{eff}+B_{i,t},P_{\max}\right)
$$

用户活跃度响应函数为：

$$
\Delta A_{ig}(r)=\alpha_g\left(1-e^{-\beta_g Point_{i,t}(r)}\right)
$$

其中，$\alpha_g$ 表示第 $g$ 类用户最大活跃提升空间，$\beta_g$ 表示第 $g$ 类用户对积分的敏感度。

参数设定方式为：

$$
\alpha_g=\rho_g(A_g^{target}-\bar{A}_g)
$$

$$
\beta_g=-\frac{\ln(1-q_g)}{P_g^{ref}}
$$

其中，$A_g^{target}$ 表示目标活跃水平，$\bar{A}_g$ 表示第 $g$ 类用户当前平均活跃度，$\rho_g$ 表示可实现比例，$P_g^{ref}$ 表示参考积分水平，$q_g$ 表示参考积分水平下可实现的活跃提升比例。

###4.5 收入、成本与利润模型

会员收入：

$$
R_m(r)=\sum_i \hat{p}_{i}^{renew,H,new}\cdot Price_{m,i}
$$

私教收入：

$$
R_p(r)=\sum_i \hat{p}_{i}^{pt,new}\cdot Price_{p,i}
$$

积分成本：

$$
C_{point}(r)=\sum_i C_i(r)
$$

平台利润：

$$
\Pi(r)=0.35R_m(r)+0.30R_p(r)-C_{point}(r)
$$

###4.6 离散策略空间与综合评价模型

设候选积分策略集合为：

$$
\mathcal{R}=\{r_1,r_2,\dots,r_m\}
$$

通过经营约束筛选可行策略：

$$
\mathcal{R}'=\left\{r_j\in\mathcal{R}:\Pi(r_j)\geq0,\ C_{point}(r_j)\leq \eta(R_m(r_j)+R_p(r_j))\right\}
$$

对每个可行策略构造评价向量：

$$
E(r_j)=\left(\Pi(r_j),\Delta A(r_j),\Delta Renew(r_j),\Delta PT(r_j),ROI(r_j),C_{point}(r_j),CostRate(r_j)\right)
$$

使用熵权法确定评价指标权重，并使用 TOPSIS 方法计算综合得分 $C(r_j)$。

最终选择：

$$
r^*=\arg\max_{r_j\in\mathcal{R}'} C(r_j)
$$

###5.3会员续费模型求解与对比结果

以 Cox 生存模型作为会员续费主模型，预测用户在会员到期后 $H$ 天内完成续费的概率：

$$
\hat{p}_i^{renew,H}=P(T_i\leq H|X_i)=1-S_i(H)
$$

同时引入 Logistic-GAM、随机森林和 XGBoost作为对比模型，统一预测目标为窗口内续费标签 $Y_i^{renew,H}$。

对比时需要说明不同模型的样本处理方式：

- Cox 生存模型可以利用窗口内续费样本、窗口内未续费样本以及未观察满 $H$ 天的右删失样本；
- Logistic-GAM、随机森林和 XGBoost 属于二分类模型，原则上只使用已经完整观察 $H$ 天的样本，即 $Y_i^{renew,H}=1$ 和 $Y_i^{renew,H}=0$ 的样本；
- 到期后尚未观察满 $H$ 天且未续费的样本不应在二分类模型中简单标记为0。

建议比较以下指标：

| 模型 | AUC | F1 | Recall | Precision | Brier Score | 是否处理右删失 |主要解释对象与说明 |
|---|---:|---:|---:|---:|---:|---|---|
| Cox 生存模型 | | | | | | 是 | 可通过风险比解释变量对续费风险率和续费时间的影响，但依赖比例风险假设 |
| Logistic-GAM | | | | | | 否 | 可通过平滑函数解释变量对窗口内续费概率的非线性影响，但不能自然处理右删失 |
| 随机森林 | | | | | | 否 | 可用特征重要性、PDP 或 SHAP 辅助解释非线性预测关系 |
| XGBoost | | | | | | 否 | 可用特征重要性或 SHAP 辅助解释复杂非线性与交互关系 |

结果分析时需要从预测性能、是否能处理右删失和模型解释对象三个角度说明主模型选择依据。即使 XGBoost 等机器学习模型在部分预测指标上略优，也应说明 Cox 生存模型更符合会员续费的时间到事件特征，因此作为主模型；Logistic-GAM 用于补充解释关键变量与窗口内续费概率之间的非线性关系，其他机器学习模型用于预测性能对比和稳健性验证。

###5.4 私教购买模型求解与对比结果

##6 模型检验与假设验证

本节用于检验模型假设是否合理，并说明模型结果的可靠性。

建议包括：

1. Cox 比例风险假设检验，例如 Schoenfeld 残差检验；
2.续费窗口 $H$ 的合理性检验，比较 $H=30$ 与 $H=60$ 下结果是否稳定；
3. 商品分类规则合理性检验，检查商品价格、周期和订单分布是否符合业务常识；
4.会员续费模型性能对比，比较 Cox 生存模型、Logistic-GAM、随机森林和 XGBoost 在 AUC、F1、Recall、Precision、Brier Score 等指标上的表现，并说明各模型在右删失处理、续费时间解释、非线性关系解释和预测性能方面的差异；
5. 私教购买模型性能对比，比较 Logistic/GAM、随机森林和 XGBoost 的预测效果与特征重要性；
6. 积分响应假设合理性说明，结合边际递减规律、文献依据和敏感性分析进行说明。

##7 灵敏度分析

本节用于检验关键参数变化是否影响最终策略选择。

建议重点分析：

1. $\alpha_g$变化对活跃提升和最优策略的影响；
2. $\beta_g$变化对策略排序的影响；
3. 积分兑换成本 $c$变化对平台利润的影响；
4. 成本收入比上限 $\eta$变化对可行策略集合的影响；
5.续费观察窗口 $H=30$ 与 $H=60$ 下结果是否稳定；
6. 熵权-TOPSIS 权重扰动后策略排序是否稳定。

可以设置三种情景：

### 保守情景

$$
\alpha_g^{low}=0.5\alpha_g,\quad \beta_g^{low}=0.5\beta_g
$$

### 基准情景

$$
\alpha_g^{base}=\alpha_g,\quad \beta_g^{base}=\beta_g
$$

### 乐观情景

$$
\alpha_g^{high}=1.5\alpha_g,\quad \beta_g^{high}=1.5\beta_g
$$

比较不同情景下的活跃度提升、续费提升、私教购买提升、积分成本、平台利润和最优策略是否稳定。
